# Проверка выпуска RAP ATLAS на Windows 10 и 11

Эту таблицу проходят на чистой виртуальной машине или отдельной тестовой учётной записи перед публикацией установщика.

## Чистая установка

- Windows 10 22H2 x64: установка без прав администратора.
- Windows 11 23H2 или новее x64: установка без прав администратора.
- Галочки «создать ярлык» и «запустить» включены по умолчанию.
- В журнале видны папка, копирование, проверка файлов, ярлыки и регистрация.
- В `%LOCALAPPDATA%\Programs\RAP ATLAS` нет файлов вне состава программы.
- В автозагрузке Windows ничего не появляется.

## Запуск и браузер

- Firefox: первый запуск, закрытие вкладки, повторный запуск.
- Firefox: назад/вперёд и возврат страницы из BFCache — heartbeat восстанавливается.
- Edge и Chrome: те же сценарии.
- Повторный двойной клик не создаёт второй процесс, а открывает уже работающий атлас.
- После закрытия последней вкладки процесс завершается.
- Запрос к `/__shutdown` без одноразового токена получает отказ.
- Другая локальная или внешняя страница не может вызвать служебные команды.

## Данные и функции

- Поиск, транслит и запрос с опечаткой.
- Фильтры «вся база / проверенные 250» и «спорные теги».
- Четыре вопроса подбора и объяснение результата без процентов.
- Три уровня карточки: коротко, продюсеру, история и связи.
- Избранное, недавние, тема, сравнение трёх карточек.
- Словарь, сообщение о неточности и редактор черновика.

## Обновление

- Установка новой версии поверх старой сохраняет данные.
- Новые файлы проверяются до замены.
- Искусственно оборванное обновление возвращает предыдущий исполняемый файл.
- Версия в свойствах EXE, списке программ, `package.json` и manifest совпадает.
- SHA-256 из `release-manifest.json` совпадает с готовым EXE.

## Удаление

- Удаление через «Установленные приложения» и ярлык из меню «Пуск».
- Журнал показывает остановку процесса, удаление ярлыков, записи Windows и данных.
- После удаления отсутствуют папка программы, ярлыки и запись в списке приложений.
- Временный деинсталлятор помечен через Windows API и исчезает после завершения/перезагрузки; скрытый PowerShell не запускается.
- Никакие соседние папки и пользовательские файлы не затронуты.
